Hexaequo - CORS Issue Help

Error: Cross-Origin Resource Sharing (CORS) Issue

You're seeing errors like:

Cross origin requests are only supported for HTTP.
XMLHttpRequest cannot load file:///path/to/model.glb due to access control checks.

Why This Happens

For security reasons, browsers restrict loading local files via JavaScript when using the file:// protocol. This is a security feature to prevent malicious websites from accessing files on your computer.

How to Fix It

To fix this issue, you need to serve your files through a web server instead of opening them directly from your file system. Here are a few simple ways to do this:

Option 1: Use Python's Built-in HTTP Server

If you have Python installed (common on macOS and Linux):

cd /path/to/Hexaequo/V2
python -m SimpleHTTPServer 8000    # Python 2
python3 -m http.server 8000        # Python 3

Then open http://localhost:8000/Hexaequo/ in your browser.

Option 2: Use the Node.js Server We Created

If you have Node.js installed:

cd /path/to/Hexaequo/V2/Hexaequo
node server.js

Then open http://localhost:8080/ in your browser.

Option 3: Use a Web Server Like Apache or Nginx

If you have a web server installed, you can configure it to serve your Hexaequo directory.

Note for Development

During development, the simplest option is usually Python's HTTP server or the Node.js server we provided.

For production deployment, you should use a proper web server like Apache, Nginx, or a cloud hosting service.

Testing Your 3D Models

Once you have a server running, you can test your 3D models by opening:

  • The model test page: http://localhost:PORT/test-models.html
  • The main game: http://localhost:PORT/index.html

Need More Help?

If you're still having issues, check:

  • That your 3D models are in the correct format (GLB)
  • That your 3D models are in the correct location (assets/models/)
  • That your 3D models have the correct names (tile_black.glb, etc.)
  • That your web server is running and accessible
